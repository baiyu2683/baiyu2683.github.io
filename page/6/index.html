[!abstract] 概述
Incus 是由 Linux Containers (LXC) 社区维护的社区驱动项目，是 LXD 的社区分支 (Fork)。它旨在提供一个完全开源、无 CLA 限制的平台，用于在单一界面下管理 Linux 系统容器 and KVM 虚拟机。

Java 泛型深度总结

Java 泛型深度总结 (Chapter 20)

[!abstract] 核心哲学
Java 泛型是为了在编译期提供类型安全检查，但在运行时通过类型擦除 (Erasure) 来保持与旧版本代码的兼容性。

1. 类型擦除 (Erasure) 的本质

在运行时，所有的泛型信息 List<T> 都会被擦除为它的原始类型（通常是 Object，或者它的边界类型）。

[!danger] 擦除带来的后果

  • 无法执行 instanceof T。
  • 无法执行 new T()。
  • 无法创建泛型数组 new T[10]。
  • 运行时 ArrayList<String> 和 ArrayList<Integer> 是同一个 Class。

2. 补偿擦除的手段

2.1 类型标签 (Type Tags)

通过显式传递 Class<T> 来找回运行时的类型感知。

2.2 工厂模式 (Factory Pattern)

利用接口避开 new T() 的限制。

  • 使用 IFactory<T>。
  • 使用 Java 8+ 的 Supplier<T>。

2.3 泛型数组的正确姿势

不要尝试直接转型 (T[]) new Object[sz]，应使用 Array.newInstance()。

3. 通配符与 PECS 原则

通配符解决了泛型的不可变性 (Invariance) 问题。

[!tip] PECS 原则 (Producer Extends, Consumer Super)

  • Producer Extends: 如果你需要一个提供 T 的容器，使用 <? extends T> (协变)。
  • Consumer Super: 如果你需要一个接收 T 的容器，使用 <? super T> (逆变)。
语法 读能力 (Get) 写能力 (Put)
<? extends T> 可以读出 T 或其父类 无法写入（除 null 外）
<? super T> 只能读出 Object 可以写入 T 及其子类

4. 进阶模式：自限定类型 (Self-Bounding)

用于在继承体系中捕获“当前子类”的类型，常见于流式 API (Fluent API)。

5. 进化：超级类型令牌 (Super Type Token)

利用匿名内部类捕获被擦除的带参数类型（如 List<String>）。这是 Gson, Hibernate 等框架实现的核心原理。

Java 泛型数组创建

Java 泛型数组创建

由于 Java 的类型擦除机制，无法直接使用 new T[] 创建泛型数组。

[!warning] 类型擦除
泛型信息在编译后被擦除，运行时 T 变为 Object，因此 new T[] 无法确定数组的实际类型。

三种创建方式

1. (T[]) new Object[size] — 内部使用

最简单的方式，适用于数组不对外暴露的场景。

[!danger] 运行时陷阱
数组实际类型是 Object[]，通过 rep() 返回后强转为具体类型（如 Integer[]）会抛出 ClassCastException。

2. Array.newInstance(Class<T>, size) — 反射方式

需要传入 Class<T> token，返回的数组运行时类型正确。

优点：数组运行时类型是 T[] 而非 Object[]，可以安全返回。

3. (Generic<T>[]) new Generic[size] — 参数化类型数组

创建泛型参数化类型的数组时，用原始类型（raw type）创建再强转。

对比总结

方式 运行时类型 需要 Class Token 可安全返回
(T[]) new Object[] Object[] 否 否
Array.newInstance() T[] 是 是
(Generic<T>[]) new Generic[] Generic[] 否 是

[!tip] 选择建议

  • 数组仅内部使用，不暴露 → 方式 1
  • 需要返回数组或保持类型安全 → 方式 2
  • 创建具体泛型类的数组 → 方式 3

三、协变与不变

数组协变（运行时检查）

数组在运行时知道元素类型，写入时检查，不匹配抛异常。

集合不变（编译时检查）

泛型设计为不变，List<Fruit> 和 List<Apple> 是完全无关的类型。

通配符实现协变（编译时安全）

编译器智能：add(E e) 中 E 是 ? extends Fruit，不确定具体是哪个子类，所以拒绝所有写入（除了 null）。但 contains(Object) 参数是 Object，可以调用。

四、PECS 原则

[!tip] PECS — Producer Extends, Consumer Super
