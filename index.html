2. 根本原因

Hexo 使用 hexo-front-matter 插件来解析文章开头的 YAML 配置。其解析逻辑如下：

  1. 如果文件以 --- 开头，则解析两个 --- 之间的内容作为 YAML。
  2. 如果文件不以 --- 开头，但正文中出现了 ---（水平分割线），解析器会尝试将第一个 --- 之前的内容作为 YAML 解析。

由于 Obsidian 用户经常在笔记顶部直接书写内容（不加 Front-matter），并使用 --- 作为视觉分割线，这会导致 Hexo 将正文内容误判为 YAML，从而引发解析错误。

PineScript_v6核心知识点

1. 语言定义与执行机制

Pine Script 是 TradingView 开发的一种轻量级脚本语言，专门用于编写技术指标和策略。

执行模型 (Execution Model)

  • 逐棒执行 (Per-bar Execution)： 脚本在图表的每一根 K 线上从左向右依次运行。
  • 计算逻辑： 每一根 K 线上，变量会根据当前及历史数据重新计算。

2. 核心数据结构：时间序列 (Series)

  • Series (序列)： Pine Script 的基本数据结构。例如 close 代表从第一根 K 线到当前 K 线的收盘价序列。
  • 回溯操作符 []： 通过 close[1] 获取上一根 K 线的收盘价，close[n] 获取历史第 n 根的数据。

3. 类型系统与修饰符

基础类型

  • int, float, bool, color, string。

类型限定符 (Qualifiers)

  • const： 编译时确定的常量。
  • input： 由用户在设置面板输入的参数。
  • simple： 在第一根 K 线确定后不再改变的值。
  • series： 随 K 线变化而变化的动态序列。

v6 新增高级类型

  • UDT (User-defined Types)： 用户定义的复合类型（对象）。
  • Enums (枚举)： 用于定义一组具名的常量集合。

4. 主要组件分类

  1. Indicator (指标)： 用于在图表上绘制线条、形状或看板，不涉及买卖模拟。
  2. Strategy (策略)： 包含买卖逻辑（strategy.entry(), strategy.exit()），支持回测和绩效分析。
  3. Library (库)： 包含可重用的函数或自定义类型，供其他脚本调用。

5. 可视化与交互工具

  • 绘图函数： plot(), plotshape(), plotchar()。
  • 动态对象：
    • line: 绘制趋势线、水平线。
    • box: 绘制矩形区域。
    • table: 创建悬浮或固定位置的数据看板。
  • Alerts (警报)： 通过 alert() 或 alertcondition() 触发实时通知。

6. v6 版本核心更新

  • 性能优化： 提高了大规模计算和绘图的执行效率。
  • 原生枚举支持： 提升了代码的可读性和参数设置的规范性。
  • 布尔值逻辑优化： 改进了布尔值在序列计算中的处理速度。
  • 增强的对象操作： 提供了更灵活的自定义对象初始化和属性访问语法。

来源： TradingView Pine Script v6 官方文档概要

数据库

学习路径

要达到“精通”水平，不能只看语法，必须深入到数据库底层原理、优化器逻辑、架构设计以及分布式理论。

2026 年，数据库领域已经从单纯的关系型扩展到了云原生和多模态。我为你精选了一份进阶书单，按学习阶段排列：

第一阶段：深挖 SQL 表达力与“奇技淫巧”

当你不再满足于简单的 CRUD，而想用 SQL 解决复杂逻辑时必读。

  • 《SQL Cookbook (第2版)》 (Anthony Molinaro)

    • 核心价值：它是 SQL 界的“食谱”。书中涵盖了大量如何用 SQL 进行行列转换、复杂报表统计、分层查询的方法。

  • 《SQL 进阶教程》 (MICK)

    • 核心价值：这本书会彻底改变你对 SQL 的认知（例如：面向集合的思维方式）。它深入讲解了 CASE 表达式、窗口函数和自连接的高阶用法。

第二阶段：性能调优与索引专家

精通 SQL 的标志是：看到一条慢 SQL，你不仅能改好它，还能解释为什么优化器选错了路。

  • 《SQL 性能解释》 (SQL Performance Explained) (Markus Winand)

    • 核心价值：强烈推荐。它不讲语法，只讲索引（B-Tree）。读完后，你会真正明白联合索引的顺序该怎么放，以及为什么 LIKE '%xxx' 会失效。

  • 《高性能 MySQL (第4版)》 (Baron Schwartz 等)

    • 核心价值：MySQL 领域的“圣经”。虽然版本更新到了第 4 版，但关于 MVCC、锁、查询优化器的核心逻辑依然是面试和实战的最高准则。

第三阶段：底层原理与架构设计（大厂必备）

要达到专家级，必须理解数据库是如何在磁盘和内存之间调度数据的。

  • 《MySQL 技术内幕：InnoDB 存储引擎》 (姜承尧)

    • 核心价值：国产精品。深度解析 InnoDB 的页结构、Buffer Pool、重做日志 (Redo Log) 等，是理解事务一致性的必修课。

  • 《数据密集型应用系统设计》 (DDIA) (Martin Kleppmann)

    • 核心价值：业内封神之作。如果你想涉足分布式数据库、时序数据库，这本书是所有分布式架构的根基。它讲解了分区、复制、事务隔离级别在分布式系统中的本质。

第四阶段：分布式与时序专题

  • 分布式：可以关注 《分布式数据库 TiDB ：原理、优化与架构设计》，它是目前国产分布式数据库中社区做得最好的，能帮你理解 Raft 协议在数据库中的应用。

  • 时序：推荐 《时序大数据平台：TDengine 核心原理与实战》。它详细介绍了如何处理每秒数百万次的写入，以及“超级表”这一创新的建模思路。

💡 专家建议的学习路径：

  1. 先读《SQL 性能解释》：这是投入产出比最高的书，能立刻解决你 80% 的查询卡顿问题。

  2. 刷 LeetCode SQL 模块（Hard 难度）：书看累了就去练手，练习窗口函数和递归查询。

  3. 看官方文档：MySQL 8.0 的 Reference Manual 是最权威的。遇到不确定的参数，文档永远是最后的真相。
