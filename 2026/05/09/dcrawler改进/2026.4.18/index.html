这是之前对项目进行改进工作的进度，继续进行
[PROJECT] 改进 Java 8 分布式爬虫 dcrawler：先补回归覆盖，再做解耦和现代化。
[STACK] Java 8 target, Maven multi-module, Spring Boot 1.4, Vert.x, HttpClient, JUnit, Fastjson, Log4j 1.x, JDK 21 build
[ARCH] CrawlerSpiderStarter 初始化 Spring/JPA/Vert.x/Jetty。 PointScheduler -> CrawlerDownloader -> CrawlerParser -> Storage。 Filter/Downloader/Storage/Processor 分模块但依赖仍耦合。

[MODULES] * dc-spider-core: Point/Config/Crawltrace/服务模型 * dc-spider-processor: 模板解析与 golden fixtures * dc-downloader-httpclient: HTTP/代理下载 * dc-storage: 存储抽象与分发契约 * dc-spider-filter: Bloom/Map filter HTTP 服务 * dc-commons: 工具、时间、编码、压缩
[DECISIONS] * 保持 source/target 1.8 → 先兼容 JDK21，不升级字节码 * tests profile 启用测试 → 保留默认 skip 行为 * Phase 2 优先 → 重构前锁定行为 * 改动保持 staged → 便于交接 [CONSTRAINTS] * 不回滚已有 staged 改动 * 不引入外部网络依赖测试 * 不做大版本依赖升级 * .codex 保持 untracked
[NEXT] 继续 Phase 2：优先诊断 weiboarticleapi2.trs 编码/emoji 期望漂移；若风险高，改补 downloader/filter 本地 mock 边界测试。
