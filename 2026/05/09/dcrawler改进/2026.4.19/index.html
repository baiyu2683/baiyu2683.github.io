[PROJECT] 改进 Java 8 分布式爬虫 dcrawler：先补回归覆盖，再做解耦和现代化。
[STACK] Java 8 target, Maven multi-module, Spring Boot 1.4, Vert.x, HttpClient, JUnit, Fastjson, Log4j 1.x, JDK 21 build
[ARCH] CrawlerSpiderStarter 初始化 Spring/JPA/Vert.x/Jetty。 PointScheduler -> CrawlerDownloader -> CrawlerParser -> Storage。 Filter/Downloader/Storage/Processor 分模块但仍有耦合。
[MODULES] * dc-spider-core: Point/Config/Crawltrace/服务模型 * dc-spider-processor: 模板解析与 golden fixtures * dc-downloader-httpclient: HTTP/代理下载 * dc-storage: 存储抽象与 TRS 契约 * dc-spider-filter: Bloom/Map filter HTTP 服务 * dc-commons: URL/时间/压缩/HKey/String 工具
[DECISIONS] * 保持 source/target 1.8 → 兼容旧运行环境 * tests profile 启用测试 → 保留默认 skip * Phase 2 优先 → 重构前锁定行为 * 改动保持 staged → 便于交接
[CONSTRAINTS] * 不回滚已有 staged 改动 * 不引入外部网络依赖测试 * 不做大版本依赖升级 * .codex 保持 untracked
[NEXT] 继续 Phase 2：补不依赖 Spring/网络/持久大文件的本地契约测试，优先 Processor 剩余 fixture 诊断或 Filter/Storage 边界；每轮跑定向 mvn -Ptests 并 git add。
