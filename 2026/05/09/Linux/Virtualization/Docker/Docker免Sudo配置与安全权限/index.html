2. ⚠️ 极其重要的安全警告 (Root Equivalence)

虽然免 Sudo 带来了便利，但必须清醒地意识到：将用户加入 docker 组，在安全性上几乎等同于给该用户提供了无密码的 Root 权限。

为什么不安全？

进入容器后，该用户可以通过 /host_root 路径直接修改宿主机上的任意敏感文件（如 /etc/shadow, /etc/sudoers），从而绕过系统的所有权限控制。

适用场景建议

  • 本地开发机： 为了方便，推荐使用此配置。
  • 多用户服务器/生产环境： 绝对不要 随意把非管理员用户加入 docker 组。
  • 更安全的方案： 如果需要极致安全，可以配置 Docker Rootless Mode（无根模式），它可以在没有 root 权限的情况下运行守护进程和容器。

3. 常见权限问题处理

如果加入组后仍然报错 Permission denied，请检查：

  1. 套接字权限： ls -l /var/run/docker.sock（所属组应为 docker）。
  2. 会话刷新： 如果 groups 命令结果中没有 docker，请务必执行 注销并重新登录。
