Java 泛型 (Generics) 核心笔记

1. 泛型的本质：参数化类型 (Parameterized Types)

泛型是将类型本身作为参数传递给类、接口或方法。其核心目标是：在编译期进行类型检查，减少运行时的类型转换异常 (ClassCastException)。

优势

  1. 类型安全：编译器会拦截错误的类型插入。
  2. 消除强制转换：代码更简洁，可读性更强。
  3. 代码重用：编写一套逻辑，处理多种数据类型（如 ArrayList<T>）。

2. 核心难点：类型擦除 (Type Erasure)

这是 Java 泛型最独特、也最受争议的特性。

原理

Java 的泛型只存在于编译期。为了兼容旧版本的 JVM，编译器在生成字节码时，会将所有的泛型参数“擦除”：

  • 如果没有边界，替换为 Object。
  • 如果有边界（如 <T extends Number>），替换为边界类型（Number）。

擦除带来的限制

  1. 运行时无法获取泛型类型：无法执行 if (obj instanceof T)。
  2. 不能创建泛型数组：无法执行 new T[10]。
  3. 基本类型不能作为泛型参数：必须使用包装类（如 List<Integer> 而非 List<int>）。

3. 通配符与 PECS 原则

这是泛型进阶中最实用的规则，决定了泛型集合的读写权限。

PECS 原则 (Producer Extends, Consumer Super)

  1. Producer Extends (<? extends T>)：

    • 场景：如果你只需要从集合中读取数据（集合是生产者）。
    • 权限：可以安全读取为 T 类型，但不能写入任何对象（除了 null）。
    • 意义：确定了上限。

  2. Consumer Super (<? super T>)：

    • 场景：如果你需要往集合中写入数据（集合是消费者）。
    • 权限：可以安全写入 T 及其子类，但读取出来只能是 Object 类型。
    • 意义：确定了下限。

4. 泛型方法与泛型类

  • 泛型类：public class Box<T> { ... } (T 在实例化类时确定)
  • 泛型方法：public <E> void process(E element) { ... } (E 在调用方法时根据参数推断)
