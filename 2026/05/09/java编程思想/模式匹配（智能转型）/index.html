🧭 场景一：带 throw（你的原始代码）

控制流图（关键）

🧠 编译器的推理

  • 走到 System.out.println(s...) 时：

    • 左边分支（true）已经 throw → 不可能走到这里

    • 只能从 false 分支来

  • false 分支意味着：

  • ⇒ s 一定存在

👉 结论：s 在这里是“确定已赋值”

路径 s 是否存在

false 分支（是 String） ✅ 有

👉 编译器无法保证 s 一定存在
