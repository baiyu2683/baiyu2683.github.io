Java 中 finally 块异常丢失及其解决方法

1. 问题背景：异常屏蔽 (Exception Masking)

在 Java 中，一个方法只能向外抛出 一个 异常。如果在 try 块抛出异常的同时，finally 块也抛出了异常，那么 finally 块中的异常会“覆盖”掉 try 块中的主异常，导致主异常丢失。

2. 解决方案 A：Try-with-resources (Java 7+ 推荐)

这是目前处理资源关闭时防止异常丢失的标准做法。Java 会自动保留主异常，并将关闭资源时的异常标记为 Suppressed（被抑制的）。

核心原理

  1. 优先抛出 try 块中的主异常。
  2. 资源关闭产生的异常会被挂载到主异常下。
  3. 可以通过 e.getSuppressed() 获取这些次要异常。

3. 解决方案 B：手动处理 (addSuppressed)

如果你必须在 finally 中编写自定义逻辑（而非简单的资源关闭），可以使用 Throwable.addSuppressed() 手动关联异常。

示例代码

4. 最佳实践

  1. 优先使用 Try-with-resources 处理所有 AutoCloseable 资源。
  2. 避免在 finally 中抛出异常：尽量在 finally 内部使用 try-catch 消化掉清理逻辑的报错，或者仅记录日志。
  3. 严禁在 finally 中使用 return：这会导致 try 块抛出的异常被无声无息地丢弃。
