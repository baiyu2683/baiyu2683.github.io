Hexo 与 Obsidian 协作集成方案

本方案旨在解决 Obsidian 与 Hexo 在图片链接语法（WikiLinks）和 Front-matter（YAML 元数据）上的不兼容问题，实现“一套笔记，双端同步”的无缝体验。

1. 核心需求

  • 图片兼容性：Obsidian 使用 ![](image.png)，Hexo 默认不识别。
  • 元数据自动化：Obsidian 笔记通常缺失 Hexo 要求的 title, date, categories 等。
  • 无损协作：不修改 Obsidian 源码的前提下，使 Hexo 能正确渲染。

2. 解决方案架构

2.1 图片渲染：原生资源支持 + 动态转换

放弃不稳定的第三方插件，采用 Hexo 原生能力与轻量级脚本结合的方式。

    作用：使 Hexo 自动处理与文章同名的资源文件夹，并正确映射图片路径。

  2. WikiLink 桥接脚本 (scripts/obsidian_bridge.js):

    作用：在渲染前动态转换语法，不影响源文件。

    • 逻辑：扫描 source/_posts，若缺失 --- 则自动补全。
