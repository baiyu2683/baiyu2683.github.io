douyin-downloader 开发改动 Checklist

这篇笔记解决什么问题

这篇笔记用于开发时快速确认：

  • 改某类功能需要碰哪些文件
  • 必须补哪些测试
  • 哪些配置、文档、桌面端同步点容易漏
  • 提交前最小验证命令是什么

  • douyin-downloader 项目开发文档

通用改动前检查

开始前先确认：

  • 当前改动是否属于共享逻辑
  • 是否需要同步桌面端项目
  • 是否会影响 Python 3.8 兼容
  • 是否在核心下载路径引入了阻塞 I/O
  • 是否会改变用户配置格式
  • 是否会改变文件命名或落盘结构
  • 是否会改变 SQLite schema
  • 是否会改变 Cookie / API key / token 等敏感信息处理

新增 URL 类型 Checklist

例如新增：

必改文件

  • utils/validators.py
  • core/url_parser.py
  • core/downloader_factory.py
  • core/api_client.py
  • core/<type>_downloader.py
  • core/__init__.py，如果需要对外导出
  • README.zh-CN.md
  • config.example.yml，如果新增配置
  • tests/test_url_parser.py
  • tests/test_downloader_factory.py
  • tests/test_api_client.py
  • tests/test_<type>_downloader.py

具体检查

  • parse_url_type() 能识别新 URL
  • URLParser.parse() 能提取稳定 ID
  • parser 返回 dict 中包含 original_url 和 type
  • downloader factory 能创建正确 downloader
  • downloader 继承 BaseDownloader
  • downloader 构造参数与其他 downloader 保持一致
  • downloader 返回 DownloadResult
  • downloader 内部主资源下载走 _download_aweme_assets() 或等价安全路径
  • 分页逻辑有 cursor 不推进保护
  • API 返回空数据时不会死循环
  • 单元测试不请求真实抖音 API

新增用户 mode Checklist

例如新增：

必改文件

  • core/api_client.py

  • core/user_modes/__init__.py
  • core/user_mode_registry.py
  • config/default_config.py

  • README.zh-CN.md
  • tests/test_user_mode_registry.py
  • tests/test_user_mode_strategies.py
  • tests/test_user_downloader_modes.py
  • tests/test_config_loader.py，如果涉及别名或默认值

Strategy 检查

  • strategy 继承 BaseUserModeStrategy
  • 设置 mode_name
  • 设置 api_method_name
  • API 直接返回 aweme 列表时不用重写 collect_items()
  • API 返回 metadata 时重写 collect_items()
  • metadata 展开可复用 _expand_metadata_items()
  • 返回列表中的每项最终包含 aweme_id
  • 去重依赖 seen_aweme_ids
  • 需要增量时补 increase.<mode>
  • 需要数量限制时补 number.<mode>

配置检查

  • number.<mode> 默认值合理
  • increase.<mode> 是否支持，不支持不要乱加
  • README 写清楚是否支持增量
  • 如果 mode 只能单独使用，要在 UserDownloader._validate_mode_scope() 中校验

修改 API Client Checklist

必改文件视情况

  • core/api_client.py
  • utils/xbogus.py
  • utils/abogus.py
  • auth/ms_token_manager.py
  • tests/test_api_client.py
  • tests/test_xbogus.py

请求检查

  • 新 API 使用 _default_query()
  • 请求 URL 走 build_signed_path()
  • 需要 live 域名时传 base_url=self.LIVE_BASE_URL
  • 需要特殊 Referer / Origin 时传 header_overrides
  • 代理参数仍传给 aiohttp
  • HTTP 429 / 5xx 有重试
  • 空 200 响应按反爬处理
  • 非 JSON 响应不会崩溃

响应检查

  • 分页接口返回 _normalize_paged_response()
  • 明确 item_keys
  • 保留 raw
  • cursor 类型转 int
  • has_more 能处理字符串和数字
  • status code 可用于诊断

测试要求

  • mock _request_json()
  • 不请求真实抖音
  • 覆盖空响应
  • 覆盖字段变体
  • 覆盖 cursor / has_more
