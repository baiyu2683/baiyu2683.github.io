项目定位

douyin-downloader 是一个 Python 3.8+ 的抖音批量下载器，核心能力是通过抖音 Web API 和 Cookie 登录态下载：

  • 单个视频
  • 单个图文 / 图集
  • 作者主页作品
  • 作者点赞
  • 作者合集
  • 作者音乐相关作品
  • 当前登录账号收藏夹作品
  • 当前登录账号收藏合集
  • 单个合集
  • 单个音乐
  • 直播录制
  • 评论采集
  • 热搜榜 / 搜索结果导出
  • 可选视频转写
  • 下载完成通知
  • 可选 REST API 服务模式

CLI 主流程

启动链路

main_async(args) 做什么

cli/main.py 的 main_async() 是整个 CLI 的调度中心。

流程：

  1. 显示 banner
  2. 确定配置文件路径，默认 config.yml
  3. 如果配置文件不存在：
    • 普通下载直接报错
    • --hot-board / --search / --serve 可以用默认配置继续
  4. 创建 ConfigLoader
  5. 应用命令行覆盖：
    • --path
    • --url
    • --thread
  6. 如果是热榜或搜索，进入 _run_discovery_subcommand()
  7. 如果是 REST 服务，进入 _run_serve_subcommand()
  8. 校验配置
  9. 初始化 CookieManager
  10. 可选初始化 SQLite Database
  11. 读取链接列表
  12. 启动 Rich 进度条
  13. 逐个 URL 调用 download_url()
  14. 汇总多个 URL 的 DownloadResult
  15. 可选发送通知
  16. 关闭数据库连接

去重逻辑

去重入口：

判断顺序：

  1. 本地文件是否已经存在
  2. SQLite 是否有 aweme 记录
  3. 如果数据库有记录但本地文件不存在，视为需要重新下载
  4. 如果本地文件存在，即使数据库没有记录，也跳过

本地去重通过扫描下载根目录中的媒体文件名实现：

只要文件名里出现 15-20 位数字 aweme_id，且文件大小大于 0，就会进入本地索引。

[!warning]
只删除数据库记录但不删本地文件，不会触发重新下载。因为本地文件扫描仍会认为该 aweme 已下载。

下载文件的可靠性

FileManager.download_file() 采用：

  1. 写入 {target}.tmp
  2. 如果响应有 Content-Length，比对实际写入字节数
  3. 字节数不一致时删除 tmp 并返回失败
  4. 成功后 os.replace(tmp, final) 原子替换

好处：

  • 避免半截文件被当成成功文件
  • 重试时不会污染正式路径

适用：

  • video
  • gallery

流程：

  2. 判断是否已下载

  4. 调 api_client.get_video_detail(aweme_id)
  5. 从详情中取作者信息

UserDownloader

适用：

  • user

流程：

  1. 从 URL 解析 sec_uid
  2. 读取 mode
  3. 校验收藏夹模式范围
  4. 调 get_user_info() 获取作者信息
  5. 为每个 mode 取 strategy
  6. strategy 收集 aweme
  7. 统一进入 _download_mode_items()

MixDownloader

适用：

流程：

  1. 从 URL 解析 mix_id
  2. 调 get_mix_aweme() 分页拉作品
  3. 调 get_mix_detail() 尝试拿作者昵称
  4. 并发下载每个 aweme
  5. mode 固定为 mix

  • number.mix 对单合集下载也生效

MusicDownloader

适用：

  • 单个音乐 URL

流程：

  1. 从 URL 解析 music_id
  2. 调 get_music_detail()
  3. 如果详情里有可播放音频 URL，下载独立音乐文件
  4. 如果没有音频 URL，回退到该音乐下首条作品
  5. 如果仍找不到，记为失败

msToken 生成

MsTokenManager 策略：

  1. 如果 Cookie 已有 msToken，直接使用
  2. 尝试读取 F2 项目的 msToken 配置
  3. 调 mssdk 接口生成真实 msToken
  4. 如果失败，生成随机 fallback msToken

注意：

  • MsTokenManager 内部有 1 小时配置缓存
  • 真实 msToken 生成失败会降级，不会中断主流程
  • 这里使用了同步 urllib.request，但在 API client 里通过 asyncio.to_thread() 调用，避免阻塞事件循环

配置校验

ConfigLoader.validate() 当前校验：

  • link 不能为空
  • path 不能为空
  • thread 必须是正整数，否则回退 5
  • retry_times 必须是非负整数，否则回退 3
  • start_time / end_time 必须是 YYYY-MM-DD，否则清空

  • max_retries 表示初次尝试后的重试次数

开发约定

Python 版本

项目要求 Python 3.8+，所以不要使用：

  • match / case
  • type X = ... 类型别名语法
  • 只在 Python 3.10+ 才安全的语法

当前代码中已有少量现代类型写法，例如 set[str]、bool | Path。如果需要严格兼容 Python 3.8，后续应统一回退到 Set[str]、Union[bool, Path]。

异步 I/O

核心路径必须使用 async I/O：

  • HTTP：aiohttp
  • 文件：aiofiles
  • SQLite：aiosqlite

不要在下载主路径里直接使用阻塞文件写入或同步 HTTP。

例外：

  • MsTokenManager 内部用同步 urllib.request，但被 asyncio.to_thread() 包住
  • tools/cookie_fetcher.py 是独立工具，不是下载热路径

常见开发陷阱

1. 只看数据库会误判是否已下载

去重同时看：

  • SQLite
  • 本地文件扫描

只删数据库不会重新下载。

2. collect / collectmix 不是任意用户收藏

它们只支持当前登录 Cookie 对应账号：

不能和普通用户模式混用。

3. allmix 仍然要兼容

虽然 canonical key 是 mix，但老配置里可能还有 allmix。

涉及 number / increase 时不要直接删除兼容逻辑。

4. 浏览器兜底只主要服务 post

browser_fallback 目前主要用于作者发布作品分页受限。

不要假设它能解决所有 like / mix / music 分页问题。

5. REST 服务不是完整桌面 sidecar

当前 CLI 的 server/app.py 是简化版：

  • 无 license
  • 无 SSE
  • 无 cancel
  • 无 overrides
  • 默认不启用数据库

不要把桌面端能力直接假设为 CLI REST 已有能力。

6. HLS 直播不是完整视频

LiveDownloader 如果选到 HLS，只保存 .m3u8 playlist。

只有 FLV 才是直接落盘的视频流。

7. 命名模板必须包含 {id}

否则同作者同一天多个作品容易互相覆盖。

8. API client 签名失败通常表现为空响应

_request_json() 中空 200 响应被视为反爬信号，会重试。

排查时重点看：

  • Cookie 是否过期
  • msToken
  • X-Bogus / A-Bogus
  • User-Agent
  • 代理
