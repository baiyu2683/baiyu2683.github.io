Incus - 系统容器与虚拟机管理

Incus: 现代系统容器与虚拟机管理器

[!abstract] 概述
Incus 是由 Linux Containers (LXC) 社区维护的社区驱动项目，是 LXD 的社区分支 (Fork)。它旨在提供一个完全开源、无 CLA 限制的平台，用于在单一界面下管理 Linux 系统容器 and KVM 虚拟机。

1. 核心定位

  • 系统容器 (System Containers): 运行完整的 Linux 发行版，表现得像物理机，但共享宿主机内核，性能几乎无损耗。
  • 虚拟机 (VM): 支持运行非 Linux 系统（如 Windows），具备完全的内核隔离。

[!important] 存储策略
默认存储在 /var/lib/incus (根目录)。如果根目录空间较小，建议将存储池迁移到较大的分区（如 /home）。

验证存储位置

  • 使用 incus storage list 查看池状态。
  • 使用 df -h /home 观察安装系统后的空间消耗。
  • 注意: /var/lib/incus/storage-pools/large-pool 是 /home 的挂载点，不会占用根目录空间。

4. 常用操作指令

操作 命令
查找镜像 incus image list images:
初始化容器（不启动） incus init images:ubuntu/24.04 my-container
启动容器 incus start <name>
初始化并启动容器(init + start) incus launch images:ubuntu/24.04 my-ubuntu
启动虚拟机 incus launch images:ubuntu/24.04 my-vm --vm
进入实例 incus exec my-ubuntu -- bash
查看状态 incus list
停止/删除 incus stop <name> / incus delete <name> --force
文件传输 incus file push/pull <src> <dest>
清理镜像缓存 incus image prune
