[Incus] 解决容器无法访问外网问题 (与 Docker 规则冲突)

1. 问题描述

在 Arch Linux 宿主机上，Incus 容器（如 ubuntu-gnome）虽然获取了 IP 地址，但无法访问外网（ping 不通公网 IP，也无法解析域名）。

2. 故障排查

在宿主机检查防火墙规则：

关键发现：

  • Chain FORWARD (policy DROP)：转发链默认策略被设置为 DROP。
  • 只有 Docker 相关的转发规则，缺乏 Incus 网桥（如 incusbr0）的放行规则。
  • 原因：宿主机安装 Docker 后，Docker 会自动修改 iptables 策略，拦截非 Docker 管理的网桥流量。
